Arrays and Hashes
9-30-2015


Arrays Hashes
  • ordered, indexed collections of objects
  • can expand as needed to hold new elements
  • can be multi-dimensional
  • key is an integer - index begins at 0
  • order determined by integer index (ascending)
  • indexed collections of objects
  • can expand as needed to hold new elements
  • can be multi-dimensional
  • key can be any object (string, integer, array, etc.)
  • order of values determined by the order keys were inserted

Arrays

The above image is a graphic representation of the array initiated by the code below:

array = ['A', 'B', 'C', 'D']

Typing array[0] would return 'A'

The array can be made up of a mixture of objects as exemplified below:

mixed_array = ['hello', 3, -5.35, [3, 'hi']]

mixed_array[0] would return 'hello'. You can also call mixed_array[3] which would return the array [3, 'hi'].

An example of a multi-dimensional array is multi_array = [['a', 'b'],['c','d']]. Calling multi_array[1][1] would return 'd'


Hashes

The above image is a graphic representation of the hash initiated by the code below:

hash = { 1 => 'A', 2 => 'B', 3 => 'C', 4 => 'D'}

Typing hash[2] would return 'B'

An example of a multi-dimensional hash is:

person = {:name => 'Nicole', :age = 21}

dog = {:name => 'Spot', :age => 3}

multi_hash = {'animal' => dog, 'human' => person}

In order to call the name of the human you would type in multi_hash['human'][:name]
